# 시나리오 비교 가이드 (Comparison Guide)

이 문서는 KB 기업카드 비대면 신청 프로세스의 4가지 주요 시나리오 간 차이점을 화면 ID(Screen ID) 수준에서 설명합니다.

---

## 1. 신용(Credit) vs 체크(Check) 비교

신용과 체크의 가장 큰 차이는 **'신용 기반 프로세스'**의 유무입니다.

| 구분 | 신용카드 (Credit) | 체크카드 (Check) | 차이점 요약 |
| :--- | :--- | :--- | :--- |
| **약관동의** | `0009` | `0009_1` | 연회비, 현금서비스 관련 약정 유무 |
| **정보동의** | `0011` | `0011` (동일) | **신용정보 조회**는 신용/체크 공통으로 동일한 의미를 가짐 |
| **금융법규** | **0018 (필수)** | **Skip (생략)** | 적합성/적정성 확인 (신용만 해당) |
| **한도설정** | **0019 (필수)** | **Skip (생략)** | 요청한도 입력 (체크는 계좌잔액 기준) |
| **결제정보** | **0022 (필수)** | **Skip (생략)** | 결제일(1~27일) 선택 기능 |
| **결과확인** | **0060 (필수)** | **Skip (생략)** | 부여된 한도 결과 확인 |

> **Key Logic**: 체크카드는 신용 평가 및 한도 협상 과정이 생략되므로 전체 시나리오가 약 3~4단계 더 짧습니다.

---

## 2. 법인(Corporate) vs 개인(Individual) 비교

법인과 개인사업자의 차이는 **'카드 소유주와 실제 사용자'**의 일치 여부에 있습니다.

### 2-1. 법인사업자 (법사자) 전용: 공용 vs 개별
법사자는 카드 결제 주체에 따라 시나리오 후반부가 달라집니다.

| 항목 | 공용 (Corporate) | 개별 (Employee) |
| :--- | :--- | :--- |
| **사용 주체** | 부서/회사 전체 공용 | 임직원 개인 소유 |
| **결제 계좌** | 법인 계좌 고정 | **개인 계좌** 연결 기능 (`0036`) |
| **개인 동의** | 불필요 | **임직원 본인 동의** 필수 (`0039`) |
| **주요 화면** | 0001 ~ 0034 (공통) | **0035 ~ 0041 (개별 전용)** 포함 |

### 2-2. 개인사업자 (개사자) 특성
개사자는 공용/개별 구분이 없으며, 사업주 본인이 곧 사용자이므로 단일 흐름으로 진행됩니다.

| 항목 | 법인사업자 (법사자) | 개인사업자 (개사자) |
| :--- | :--- | :--- |
| **시작 구간** | `0001 ~ 0008` (공용) | `0001` (이후 전용 화면 점프) |
| **전용 화면** | 기존 화면군 활용 | **`0053 ~ 0074` (개사자 전용)** |
| **본인 인증** | 법인 공동인증서 (`0032`) | 개인 3종 본인인증 (`0058`) |

---

## 3. 화면 ID별 핵심 체크리스트

각 화면에서 신용/체크 여부에 따라 달라지는 로직입니다.

*   **0018 (필수확인사항)**: 연소득, 부채, 신용점수 구간 등 **신용 평가용 질문**이 집중됨.
*   **0019 (이용한도)**: "카드사가 얼마까지 빌려줄까?"를 묻는 신용카드 전용 단계.
*   **0022 (결제일)**: 매달 카드값을 갚을 날짜를 지정하는 신용카드 전용 단계. (체크는 즉시출금)
*   **0036 (개인계좌)**: 법인 소속 사원이 카드 대금을 **자기 통장**에서 빠져나가게 설정할 때 사용. (법인 개별카드 전용)
